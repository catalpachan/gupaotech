科技公司官网纯静态前端 HTML/CSS 需求文档

一、项目概述

1. 项目名称
科技公司官网首页静态展示页

2. 项目目标
基于提供的官网原型图，输出一套纯静态前端页面，用于官网首页展示。页面仅实现视觉呈现与基础悬停反馈，不涉及后端接口、数据库、登录鉴权、内容管理系统、表单提交、复杂交互逻辑或 JavaScript 动态数据渲染。

3. 开发范围
本项目仅包含：
- HTML5 页面结构搭建
- CSS3 页面样式开发
- 响应式基础适配
- 基础 hover 视觉反馈
- 品牌视觉统一规范落地

不包含：
- Vue / React / Angular 等框架开发
- JavaScript 业务逻辑
- 接口联调
- 后台管理系统
- 动画特效开发
- SEO 深度配置
- 多语言切换
- CMS 内容维护

二、设计定位与视觉风格

1. 品牌调性
整体风格定位为：
- 现代简约
- 轻量化
- 科技感
- 玻璃质感
- 清晰有序
- 具备官网品牌展示属性

2. 品牌色要求
网站主品牌色使用橙色系，建议作为主按钮、标签高亮、导航选中、重点数据强调、分割装饰线等核心视觉元素的主色。

建议品牌色参考值：
- 主品牌橙：#FF7A1A
- 悬停橙：#FF8F3D
- 浅橙背景：#FFF3E8
- 橙色描边：#FFD1B0

3. 辅助色建议
- 主文字：#1F1F1F
- 次文字：#666666
- 弱文字：#999999
- 页面背景：#F7F8FA
- 模块白色：rgba(255,255,255,0.72)
- 边框颜色：rgba(255,255,255,0.45)
- 阴影颜色：rgba(31,31,31,0.06)

4. 玻璃质感表现方式
页面需体现轻玻璃拟态效果，建议通过以下 CSS 方式实现：
- 半透明白色背景
- 柔和描边
- 模糊背景层次
- 微阴影
- 大圆角卡片
- 局部高光与渐变叠加

视觉关键词：
- backdrop-filter: blur()
- 半透明卡片
- 柔和投影
- 圆角 16px / 20px / 24px
- 大留白
- 信息层级明确

三、技术实现要求

1. 技术栈
仅使用以下技术：
- HTML5
- CSS3

禁止使用：
- JavaScript
- jQuery
- 任意前端框架
- UI 组件库
- CSS 预处理器（如 Sass/Less，可选但非必须，最终需输出纯 CSS）
- 构建工具依赖

2. 页面要求
- 页面结构语义化清晰
- 样式模块化命名规范
- 代码具备良好可读性
- 兼容主流现代浏览器
- 支持桌面端优先展示
- 可做基础响应式适配平板与移动端

3. 布局方式
建议使用：
- Flex 布局
- Grid 布局
- 容器居中布局
- 分区块模块化开发

页面推荐最大内容宽度：
- 1200px 或 1280px

四、页面整体结构说明

首页建议划分为以下模块：
1. 顶部导航区
2. 一级分类标签区
3. Banner / 背景主视觉区
4. AI 行业价值说明区
5. 核心数据展示区
6. 政策/趋势文案说明区
7. 学习价值与职业发展引导区
8. 课程主推荐区
9. 课程卡片展示区
10. 学员案例展示区
11. 案例卡片列表区
12. 页脚信息区

五、各功能模块详细描述

1. 顶部导航模块
模块作用：
用于展示品牌 Logo、一级导航入口，承担官网全局信息导航功能。

内容构成：
- 左侧公司 Logo
- 右侧顶部导航菜单

导航栏目建议：
- 首页
- 产品服务
- 企业方案
- 社区生态
- 资讯动态
- 品牌故事
- 加入我们
- 联系我们

样式要求：
- 白色/半透明玻璃背景
- 导航整体横向排列
- 当前页高亮显示
- hover 时字体变橙色或底部出现橙色细线
- Logo 区域视觉清晰，建议左对齐

前端说明：
- 使用 header 包裹
- 导航使用 nav + ul + li + a
- 当前选中项增加 .active 样式类

2. 一级分类标签模块
模块作用：
用于展示当前业务板块分类，如 AI 系列、IT 职场等，承担频道切换视觉表达功能。

内容构成：
- 左侧大类标题，如“AI系列”
- 右侧大类标题，如“IT职场”
- 下方分类 tab，例如：
  - AI通识
  - AI应用
  - 论文辅导
  - 竞赛指导

样式要求：
- tab 横向排列
- 当前选中 tab 使用橙色高亮
- 卡片边框清晰，风格简洁
- 标签具备玻璃质感底色或浅白色块

前端说明：
- 使用列表形式实现
- 仅保留静态选中态，不做点击切换逻辑
- 支持 hover 样式

3. Banner 主视觉模块
模块作用：
作为首页核心视觉区，承载品牌主张、业务定位与第一屏重点信息传达。

内容构成：
- 大标题文案
- 副标题文案
- 背景图占位区域
- 可预留品牌插画/科技背景图位置

示例文案方向：
- 顺应时代浪潮，铸就 AI 人才基石
- 响应国家战略，为智能时代的国家竞争力筑基

样式要求：
- 模块面积较大
- 使用浅色渐变或玻璃卡片底
- 中心对齐排版
- 背景可使用科技感线性渐变、光晕、网格、模糊装饰
- 可预留图片占位框但不做真实轮播

前端说明：
- 纯静态结构
- 不做自动轮播
- 不做视频背景
- 可使用 CSS 渐变模拟科技氛围

4. 行业价值说明模块
模块作用：
用于强调 AI 行业趋势、国家战略背景与职业发展价值。

内容构成：
- 大段标题
- 分层小标题说明

示例结构：
- 从国家战略力到职场竞争力
- AI 已成为定义未来十年发展格局的核心引擎

样式要求：
- 字号层级明显
- 左对齐排版
- 行间距舒适
- 保持页面节奏感与留白

5. 核心数据展示模块
模块作用：
以数据卡片形式展示行业规模、趋势指数、排名等关键信息，提高说服力与视觉节奏。

内容构成：
建议 4 个横向数据卡片，例如：
- 引领发展新质生产力
- 超 9000 亿
- 2.30
- 第一位

每张卡片包含：
- 主标题
- 核心数字
- 简短说明

样式要求：
- 横向四列布局
- 卡片统一高度
- 白色玻璃卡片风格
- 数字重点突出
- hover 时卡片轻微上浮或阴影增强（仅 CSS）

前端说明：
- 使用 grid 或 flex
- 支持响应式折行

6. 政策/趋势说明文本模块
模块作用：
用于承载较长篇幅的行业趋势说明、政策解读、数据来源说明等静态文本内容。

内容构成：
- 编号段落说明
- 文字内容较多

样式要求：
- 文本区域宽度适中
- 字号适中，阅读舒适
- 段间距清晰
- 可加入左侧编号样式或浅色分割线

前端说明：
- 使用静态文本段落即可
- 不做展开收起
- 不做富文本编辑功能

7. 学习价值与职业引导模块
模块作用：
用于引导用户理解系统化学习 AI 的价值，建立课程内容与职业成长之间的联系。

内容构成：
- 主标题
- 强调性副标题
- 中心价值文案

示例方向：
- 系统化学习并掌握人工智能相关知识与技能
- 把握未来职业发展机遇，顺应国家发展战略需求

样式要求：
- 模块整体居中
- 标题强调感更强
- 可加入橙色细线、图形点缀、淡背景光晕

8. 课程主推荐模块
模块作用：
用于突出一个重点课程类别，例如“AI通识”，并给出该类课程的核心介绍。

内容构成：
- 左侧课程分类标签
- 右侧课程介绍文案

示例内容：
- AI通识
- 旨在系统化培养具备前沿视野与实战能力的人工智能专业人才……

样式要求：
- 左右双栏布局
- 左侧标题标签具备重点识别度
- 右侧文案清晰、层级合理
- 玻璃卡片容器包裹

前端说明：
- 纯静态文本展示
- 可预留图标位置

9. 课程卡片展示模块
模块作用：
用于展示具体课程条目，引导用户进一步了解课程详情。

内容构成：
建议 3 张课程卡片，例如：
- L4：深度学习课程封面
- L4：机器学习课程封面
- L4：计算机视觉课程封面

每张卡片包含：
- 课程等级/标签
- 课程名称
- 简短说明
- “了解课程详情”按钮

样式要求：
- 三列等宽布局
- 卡片风格统一
- 按钮使用品牌橙
- 卡片底部留出按钮区
- 鼠标悬停时卡片阴影增强

前端说明：
- 不做点击跳转逻辑
- 按钮仅保留 a 标签展示状态
- 封面图可用占位图或 CSS 图形块代替

10. 学员案例展示模块
模块作用：
展示学员成长成果与真实案例，提高页面可信度和转化能力。

内容构成：
- 模块主标题
- 简短介绍文案

示例方向：
- 学员真实案例：看得见的成长
- 用真实项目成果、高水平论文、成功就业与职业转型，书写成长篇章

样式要求：
- 标题显著
- 区域背景与上下模块有层次区分
- 简介文字不宜过长

11. 案例卡片列表模块
模块作用：
展示多个案例内容摘要，作为官网内容模块的重要组成部分。

内容构成：
建议每张案例卡片包含：
- 案例标题
- 简述内容
- 图片占位区域
- “点击了解更多”按钮

样式要求：
- 两列或三列布局
- 图片区域可用占位图
- 卡片风格统一
- 保持现代简约，不要过度装饰
- 按钮需与品牌色统一

前端说明：
- 静态展示，无轮播
- 图片可先使用灰色占位框
- 可预留后期替换真实案例图位置

12. 页脚模块
模块作用：
展示官网底部信息，补充品牌信息与基础链接。

内容构成建议：
- 公司名称
- 简短品牌介绍
- 联系方式
- 地址信息
- 版权信息
- 备案信息
- 页脚快捷链接

样式要求：
- 背景颜色较主体略深
- 内容排版简洁
- 字体颜色偏弱化
- 与整体风格统一

六、交互与状态说明

虽然本项目为纯静态 HTML/CSS 页面，但需具备基础视觉状态反馈：

1. Hover 状态
适用于：
- 顶部导航
- Tab 标签
- 按钮
- 数据卡片
- 课程卡片
- 案例卡片

2. 视觉反馈建议
- 字体颜色变化
- 阴影增强
- 边框高亮
- 微小上浮
- 背景色轻微变化

3. 禁止事项
- 不使用 JavaScript 动效
- 不做轮播
- 不做手风琴
- 不做弹窗
- 不做表单校验

七、响应式适配要求

1. 适配范围
建议至少兼容以下尺寸：
- 桌面端：1440px、1280px、1200px
- 平板端：1024px、768px
- 移动端：可做基础堆叠适配

2. 响应式策略
- 导航在小屏下可自动换行或简化处理
- 多列卡片在中小屏改为两列或单列
- Banner 和数据模块需保持排版完整
- 保证文字不拥挤、不重叠

八、页面输出要求

1. 文件结构建议
/index.html
/css/style.css
/images/...

2. 输出规范
- 一个完整首页 HTML 文件
- 一个独立 CSS 文件
- 图片资源统一路径管理
- 注释清晰
- 类名语义化

3. 命名建议
推荐使用 BEM 风格或统一语义化命名，例如：
- header
- nav
- hero
- section-title
- stats-card
- course-card
- case-card
- footer

九、验收标准

项目完成后需满足以下要求：
- 页面结构与原型逻辑一致
- 所有模块完整呈现
- 品牌橙色体现明确
- 整体风格符合现代简约、轻量化、玻璃质感
- 代码仅使用 HTML/CSS
- 页面在主流浏览器中显示正常
- 基础 hover 状态完整
- 页面视觉整洁、层级清晰、留白合理
